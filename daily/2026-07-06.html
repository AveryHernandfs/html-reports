Learning to Code Is Still Worthwhile
Steve Krouse（ValTown 创始人）反驳"AI 时代不用学编程"的观点。核心论点：AI 让编码更容易，但理解系统的能力反而更稀缺。编程教育的价值从"写代码"转向"理解抽象和系统思维"。
在 AI coding 工具（Claude Code、Cursor 等）普及的今天，很多人认为学编程已经过时。但 Krouse 认为，AI 降低了编码门槛，却提升了"理解系统"的门槛。能读懂代码、调试问题、设计架构的人会更值钱。这和 USER.md 里你正在做的事（用 AI agent 写代码但理解底层）完全一致。

Companies Hire More After AI Adoption
Ramp 数据显示：重度 AI 采用的公司不裁员反而扩招。AI 不是替代人，而是让现有团队能做更多事，从而需要更多人来处理增长的业务量。
Ramp（企业支出管理平台）分析了数千家客户数据，发现 AI 采用率高的公司招聘量反而增长。这与"AI 会大规模失业"的叙事相反。关键洞察：AI 提高了生产力 → 业务增长 → 需要更多人。对创业公司来说，这是一个积极信号。

Kani: A Model Checker for Rust
Kani 是 Rust 的形式化验证工具（model checker），可以证明 Rust 程序满足指定的安全属性。基于 CBMC，支持验证unsafe代码的内存安全。

Clojure 1.13 Adds Support for Checked Keys
Clojure 1.13 引入 checked keys：在 map 结构中强制检查 key 的存在性和类型。从动态语言向"渐进式类型安全"迈出重要一步。

矩阵乘法（Matrix Multiplication）
矩阵乘法不是对应元素相乘，而是"行乘列"：
C = A × B，其中 C[i][j] = Σ A[i][k] × B[k][j]

直觉理解：矩阵乘法 = 多个线性变换的组合。先做 B 变换，再做 A 变换，结果就是 A×B。
| a₁₁ a₁₂ | | b₁₁ b₁₂ | | a₁₁b₁₁+a₁₂b₂₁ a₁₁b₁₂+a₁₂b₂₂ |
| a₂₁ a₂₂ | × | b₂₁ b₂₂ | = | a₂₁b₁₁+a₂₂b₂₁ a₂₁b₁₂+a₂₂b₂₂ |
练习：如果 A 是 2×3 矩阵，B 是 3×4 矩阵，那么 A×B 的结果是几行几列？
✅ 正确！矩阵乘法的结果维度 = A的行数 × B的列数。2×3 × 3×4 = 2×4。中间的 3"消掉"了。
❌ 不对。矩阵乘法结果维度是 A的行 × B的列，不是取公共维度。
❌ 不对。2×3 × 3×4 的结果不是 2×3。
❌ 不对。3×4 是 B 的维度，不是结果的维度。
